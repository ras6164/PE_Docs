Quick Links

Logging into your Site

  • Navigate to www.yourdomainhere.org/wp-admin
  • Input your username/email address and password and click the blue "Log In" button
  • The dashboard will now be loaded. This is the backend of your website. This page shows basic website information, but it is not often used besides when logging in.
  • The grey sidebar on the left side of the screen is where you primarily will be using your site

Adding New Users/New Administrators

  • The main purpose of adding users is if you want to grant someone access to the backend of the site and assist/handle/manage the users, pages, events, or anything else administrative on the website
  • Navigate to www.yourdomainhere.org/wp-admin
  • Once logged in, click on "Users" in the grey sidebar
  • All users will be displayed in a table on this page.
  • To add a new user, click the "Add New" Button in the top left of the screen.
  • A form to enter in user information will now be displayed. Input username and email, and save the password for later or change it to something easier here, after clicking “Show Password”.
  • To give  users full access to the site, in the “Role” drop down box, select “Administrator”
  • Once you are completed inputting information, click the “Add New User Button”

Embedding Model Viewer Resources 

  • Navigate Model Viewer's Editor (modelviewer.dev/editor/).
  • Upload gITF or GLB file to and input settings as desired on side panel.
  • Copy code snippet.
  • Add component scripting line to import the modeling engine.
    • <!-- Import the component --> <script type="module" src="https://unpkg.com/@google/model-viewer/dist/model-viewer.min.js"></script>
  • Wrap the copied code output from Model Viewer in a new div that styles width and height to 100%. Highlighted code is added from Model Viewer Output.
    • <div style="width:100%; height:100%"> <model-viewer style="width:100%; height:100%" src="https://modelviewer.dev/shared-assets/models/NeilArmstrong.glb" ar ar-modes="webxr scene-viewer quick-look" camera-controls environment-image="neutral" poster="poster.webp" shadow-intensity="1" auto-rotate> </model-viewer></div>
  • Change the source in the Model Viewer output to match the source of the .GLB/.gITF file hosted on your public server.
  • Navigate to the Avada page/post you wish to add the model to
  • Add a container/column and add select "Code Block"
  • Copy the Import Component and edited Model Viewer Output with Div added and save changes.

Utilizing Contact Forms

The Contact Form automatically sends all inputs via email to TBD and a stored database on the backend of the website. In order to access the database:

  • Navigate to www.yourdomainhere.org/wp-admin
  • Once logged in, Navigate to "Avada" on the grey sidebar.
  • Select Forms
  • To view entries, click "view entries" under the entries column for the respectful form (ex. contact form).
  • To edit forms, click the name of the form you want to edit, and it will take you into the editor.

Editing Pages

Your website makes use of a builder known as Avada. It features two editors: traditional and live view. The traditional editor makes use of a standard WordPress block editor. The live editor is much more interactive and allows one to edit directly on the frontend of the website.

  • Navigate to www.yourdomainhere.org/wp-admin
  • Once logged in, navigate to "Pages" on the grey sidebar.
  • Find the page one wishes to edit.
  • To use the traditional editor, simply click on the page name
  • To use the live editor, hover over the page name one wishes to edit, and click "Avada Live."

Adding News

  • Navigate to www.yourdomainhere.org/wp-admin
  • Once logged in, navigate to "Posts" on the grey sidebar.
  • Edit/Add New News Posts by utilizing this page. All news section of the website are fed from this section. Do not post projects here unless you want them to display with the news section.

Adding Projects

  • Navigate to www.yourdomainhere.org/wp-admin
  • Once logged in, navigate to "Portfolio" on the grey sidebar.
  • Edit/Add New Projects by utilizing this page. All project posts of the website are fed from this section. For a more familar editing style, you can utilize the "Avada Live" editor to add custom content and code block to these pages. It is important to make the Portfolio Category on the right column before publishing or else it will not populate on the website.

Updating WordPress Plugins

You may have to update your website from time to time, and it is important to keep your plugins/resources updated. For the most part, you should be able to operate the site without updating any plugins but there are inherent security risks with using out of date software.

  • Navigate to www.yourdomainhere.org/wp-admin
  • Once logged in, click on "Plugins" in the grey sidebar towards the bottom.
  • The plugins page will now be displayed.
  • In most cases, click the blank check mark box on the top of the table.
  • Then click the “Bulk Actions” drop down box above the table
  • Then select “Update” from the drop down box, and click the “Apply” button
  • It may take a few minutes to update / respond. The table will reflect if an update has occurred
  • You have now successfully updated your website plugins!
